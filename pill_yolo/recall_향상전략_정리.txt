============================================================
알약 각인 Detection — recall 향상 전략 정리
============================================================

[현재 상황]
- 환경: GTX 1050 (2GB VRAM), 706장 train
- 모델: YOLO26n + P2 head, imgsz=960, batch=4, workers=2
- 설정: freeze=10 (이후 freeze=0 비교), aug erasing=0.4 / mosaic=0.5 등
- 결과(plateau):
    precision 0.80 / recall 0.30 / mAP50 0.27 / mAP50-95 0.08
    (freeze=10: ep104 EarlyStop, best ep74)
- 이미 시도: freeze=0(차이 미미), 라벨 16장 QA(양호),
  augmentation 완화는 다음 계획

[핵심 진단]
- precision 高 / recall 低 = "찾은 건 맞는데 70%를 놓침"
- mAP50 0.27 vs mAP50-95 0.08 폭락 = 전형적 small-object 신호
  (작은 박스는 몇 픽셀만 어긋나도 IoU 붕괴)
- freeze·augmentation은 효과 약했으니 우선순위 아님


============================================================
1단계 — 더 학습 돌리기 전에 진단 (모두 cheap)
============================================================

(1) 클래스별로 분리해서 보기
  - 현재 recall은 imprint + score_line 합산
  - score_line은 드물고 가늘어 recall 바닥일 가능성 → 전체를 끌어내림
  - model.val() per-class 확인
  - 권장: detection은 single_cls(imprint만)로 단순화
    → capacity 집중으로 recall 상승 기대
    → score_line 구분은 다운스트림 lexicon으로 처리

(2) PR 커브 / 낮은 confidence에서의 recall
  - Ultralytics P/R은 best-F1(운영 임계값) 기준
  - model.val(conf=0.001)로 '최대 recall' 확인
  - 만약 거기서 recall 0.8 나오면 = 모델은 볼 수 있고 confidence만 낮은 것
    → STR이 FP 걸러주므로 그냥 낮은 conf로 운영하면 해결
    → 가장 싸게 끝날 수 있는 시나리오, 제일 먼저 확인

(3) 평가 지표 자체 재검토 (중요)
  - COCO instance mAP는 실제 목적보다 가혹
  - 진짜 필요한 건 "각인 있는 이미지마다 STR로 넘길 만큼 영역을 잡았나"
    = 이미지 단위 recall (tight IoU 아님)
  - 예측∪ vs GT∪ 커버리지로 측정하면 실제 유용성은 0.30보다 훨씬 높을 수 있음
  - 이 지표로 바꾸면 "성능 나쁨" 진단 자체가 바뀔 수 있음


============================================================
2단계 — 근본 원인 공략: "각인이 너무 작다"
============================================================

(A) 탐지 전에 알약 영역으로 먼저 crop  ★ 2GB 환경 최대 레버
  - 알약 중앙 정렬됨 + autolabel 코드에 pill_bbox() 이미 있음
  - 알약 외곽 crop 후 그 안에서 각인 탐지
  - 각인 픽셀 비율↑ → small-object 문제 직접 완화 (유효 해상도↑ 효과)

(B) SAHI 등 타일 추론
  - 추론 시 타일 분할→탐지→병합
  - 학습 VRAM 안 늘리고 작은 객체 recall 크게 상승
  - (A)와 결합 시 더 효과

(C) 2GB GPU가 사실상 진짜 병목  ★ 단일 임팩트 최대 가능
  - batch=4 → BatchNorm 통계 noisy → 성능 천장 누름
  - imgsz 1280 올리고 싶어도 VRAM이 막음
  - 무료 Colab/Kaggle T4(16GB)로 옮겨 batch=16~32 @ imgsz=1280 한 번 제대로
  - 2GB에서 10시간 실험 반복보다 빠르고 정확한 비교


============================================================
3단계 — 데이터와 초기화
============================================================

더 많은 라벨
  - recall은 데이터 양에 거의 비례. 706장은 detection엔 작음
  - autolabel 더 돌리기 + AI Hub 경구약제 데이터 합치기
  - QA를 16장(2%)만 함 → 박스 '일관성'을 더 넓게 확인
    (Gemini consensus 박스 tight 정도가 들쭉날쭉하면 일관된 경계 못 배워
     recall·mAP 동시 정체) → 박스 크기·종횡비 분포 히스토그램으로 점검

COCO 대신 텍스트 친화 초기화
  - "COCO vs 각인 도메인 다름"의 해법은 freeze 풀기보다 더 나은 초기화
  - scene-text detection 가중치 / 합성 텍스트 사전학습 backbone이
    COCO보다 훨씬 잘 전이


============================================================
우선순위 정리
============================================================

지금 당장 (cheap 진단):
  ① per-class 분리 + single_cls(imprint only)
  ② conf=0.001 recall 확인
  ③ 이미지 단위 커버리지 지표로 재평가
  → 여기서 "사실 쓸 만함" 또는 "임계값만 낮추면 됨"으로 끝날 수 있음

그게 아니면 (구조적 레버):
  ④ 알약 crop 후 탐지
  ⑤ Colab T4로 batch 키워 재학습
  ⑥ 데이터 증량(AI Hub) + 박스 일관성 QA

후순위:
  - augmentation 완화: 작은 객체에서 mosaic↓는 합리적이나
    위 레버들보다 후순위
  - freeze 조정: 이미 효과 약함 확인됨
